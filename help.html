COMMAND.COM, CMD, TCC, TCC/LE

White letters on black background - are you confused?

Why not?

If you don't like this window, if you don't feel comfortable with the keyboard instead of mouse - command processors (including TCC/LE) are not designed for you.

So why?

If you are experienced computers user, professional, admin, programmer, linux user familiar with shells like bash - you will find TCC/LE interesting.

TCC (from JP Software) is a full replacement of standard CMD.EXE and COMMAND.COM. It emulates all the behavior of Windows' shell, including CMD errors, undocumented features and less known semi-variables expansion. All BATches, all commands can be executed using TCC shell, leading to the same results. One of key factors of success is compatibility. There is no word "obsolete" in TCCs documentation, all old BATch scripts can be successfully executed. TCC is a good replacement of windows shell.

TCC/LE is a "Light Edition" of TCC. There are removed dozens of command and options, but TCC/LE is still compatible with CMD.EXE. TCC/LE is not enough to perform advanced administrators tasks. For example: TCC/LE is unable to monitor CPU usage and notify you about high usage using Jabber protocol. TCC/LE is a powerful full shell for personal daily use.

Free

And last, but not least: TCC/LE is available without any fees. It is not a free/open software, the code is closed and not available to public, it is commercial product. But it fills the gap, there is nothing comparable to TCC/LE. TCC/LE introduces about 80 new internal commands, about 80 new variables, provides more than 200 built-in functions, adds new switches to well known commands (for example: there is 15 options for standard DIR, while TCC/LE's DIR command allows 30 options, date and size ranges, you can put dir results to the clipboard and there is programmable DIR command: PDIR. Maybe I forgot some features.)

Not just another portable app.

The whole subsystem

Kilometers driven by mouse, millions of clicks...

Portable TCC/LE is designed for command-line maniacs. If you like keyboard, if you can agree, that looking for mouse is a waste of time - you could like TCC/LE. You can add aliases for your favourite commands. You can add portable command line tools (java, python, perl, busybox) to get portable, machine independent and ready to use environment.

TCC/LE power

4DOS (the first member of the family) was re lased in 1989, and it is still actively developed. For comparison: Linux was released in 1991. The expression of TCC and bash scripts are at least (in my opinion) comparable.

Current version of TCC is 15.0, current version of TCC/LE is 13.0

At first: take a look at TakeCommand and TCC specification. Please ignore TakeCommand' parts (it is just another member of the family), please focus on TCC and TCC/LE. Take a look at aliases, "variable functions" (things starting with "%@"), SWITCH and DO statements (not so frequently used). Do you feel the power?

Portable Apps flexibility

During initialization process startup script scans existing portable applications and performs additional steps - modifies environment (including %PATH% variable), deactivates conflicts (for example between Java installed on the host machine and Portable Java), adds app-specific aliases. It even adds names of existing apps as aliases, so you can, for example, start FireFox directly from command line entering "Firefox <Enter>". And more...

Customizations, configurations

There are available three levels of customizations:

  • TCC/LE settings, defined in TCC/LE configuration and OPTION command,
  • TCC/LE Portable launcher behavior, described below,
  • Optional TCC/LE Portable customizations, making use of PortableApps environment, described below,

TCC/LE settings

Most of the shell settings are stored in TCC.INI. The content of the file is mostly portable, its location is maintained by TccLePortable launcher. Several non portable settings (location of changeable files or directories) are derived from TccLePortable.ini and are maintained by the launcher.

Launcher location

TccLePortable.exe can be located in the directory TccLePortable (standardized location of portable apps launcher) and one (..\TccLePortable) or two levels (..\..\TccLePortable) above, so TCC/LE launcher can be stored in the same directory, where Start.exe exists.

Location of TccLePortable.ini doesn't depend on the location of the launcher itself. TccLePortable.ini is expected to be found in the directory TccLePortable. If there is no INI file it is created in the proper location.

Launcher Configuration

TccLePortable.ini is expected to be found in the directory TccLePortable. Some settings are specific to core TCC/LE settings, some options are used in portable customizations.

All settings are stored within [TccLePortable] section.

[TccLePortable]

SettingsDirectory    =Data\settings
TccIniFile           =tcc.ini
AdditionalParameters =/D

Editor               =

LogsDirectory        =Data\logs
LogName              =command.log
LogErrorsName        =errors.log
HistLogName          =history.log

ProfileDirectory     =Data\profile
DirHistFile          =DirHistory.dat
HistFile             =History.dat

TempDirectory        =Data\temp

Force32              =true

ShowSplash           =true
DirDrivesFile        =DirDrives.dat

SettingsDirectory (affects [4NT]4STARTPath [4NT]4StartPath in tcc.ini): contains start up files tcc.ini, 4START and 4EXIT.

TccIniFile and AdditionalParameters are passed as parameters during execution of tcc.exe.

Editor (affects [4NT]Editor in tcc.ini) (rarely used): - the path name of the editing program to run from the LIST command.

LogsDirectory, LogName, LogErrorsName, HistLogName (affect [4NT]LogName [4NT]LogErrorsName [4NT]HistLogName in tcc.ini): are used to store TCC logs (if enabled with OPTION command).

ProfileDirectory (affects [4NT]DirHistFile [4NT]HistFile [4NT]TreePath in tcc.ini): There are some machine/account specific settings (history, directory history, JpsTree.idx). Content of that files is not portable, due to absolute path used. All the non portable files are stored in the separated directories: ProfileDirectory\MachineName\UserName.

DirHistFile (together with ProfileDirectory affects [4NT]DirHistFile in tcc.ini): TCC/LE loads the specified directory history list file before running 4START, and saves the directory history to the file after running 4EXIT. Hint: you can define DirHistFile as ..\DirHistory.dat to get machine specific, but common to all accounts list, or you can define DirHistFile as ..\..\DirHistory.dat to get common list (machine/account independent).

HistFile (together with ProfileDirectory affects [4NT]HistFile in tcc.ini): TCC/LE loads the specified command history list file before running 4START, and saves the command history to the file after running 4EXIT. Hint: you can define HistFile as ..\History.dat to get machine specific, but common to all accounts list, or you can define HistFile as ..\..\History.dat to get common list (machine/account independent).

TempDirectory: Launcher sets %TEMP% and %TMP% variables to that value before tcc.exe starts. TEMP/TMP settings are inherited by all programs executed from command line. Leaving this setting empty (no value) prohibits launcher environment manipulation and tcc.exe uses users settings.

Force32: Value "true" (default) forces launcher to start 32-bit version of tcc.exe always (file App\TCC_32\tcc.exe). Any other value allows to start 64- bit version on 64-bit platforms (file App\TCC_64\tcc.exe)

ShowSplash (it has different meaning, than PortableApps' ShowSplash settings) used within customizations: Custom 4START displays information gathered from plugins - plugin name and one-line comment. ShowSplash=false forces start more quiet.

DirDrivesFile used within customizations: Custom 4START loads current directory for all drives from that file and 4EXIT stores current directories there. Name of file is combined with ProfileDirectory.

TCC.INI [Portable]

Some run time info, specific to Portable configuration are stored with tcc.ini file in [Portable] section and are available for 4START_NEXT/4EXIT_NEXT (using function %@IniRead, where full name of tcc.ini can be found in variable %_ININAME. Most of them are derived from TcclePortable.ini.

[Portable]
LocalConfig=
LocalConfigMachine=
LocalConfigUser=
DirDrivesFile=
ShowSplash=
Force32=
TccLePortable.ini=
TccLePortable.exe=

LocalConfig: Full name of ProfileDirectory

LocalConfigMachine: Full name of ProfileDirectory\MachineName

LocalConfigUser: Full name of ProfileDirectory\MachineName

DirDrivesFile: Value of DirDrivesFile

ShowSplash: Value of ShowSplash

Force32: Value of Force32

TccLePortable.ini: Full file name of launcher configuration file TccLePortable.ini

TccLePortable.exe: Full file name of launcher executable TccLePortable.exe

Directory structure

TCC/LE directory structure, files appinfo.ini and installer.ini conforms to PortableApps.com Format™ 3.0 specification.

Data\

There are sub-directories of Data\, described before:

  • Data\temp: pointed by TempDirectory and environment variables %TEMP% and %TMP%.
  • Data\logs: pointed by LogsDirectory, contains error log, command log and history log files (if enabled).
  • Data\settings: pointed by SettingsDirectory, contains configuration file tcc.ini, 4START, 4EXIT and 4START_NEXT/4EXIT_NEXT scripts.
  • Data\profile: pointed by ProfileDirectory, contains MachineName\UserName sub-directories and environment specific files (history, directory history, jpstree.idx, current directories for all drives).

All names of Data\ sub-directories can be changed within TccLePortable.ini.

App\

There are some important sub-directories of App\:

  • Data\TCC_32: contains all 32-bit binaries of TCC/LE and help files. App\TCC_32\tcc.exe is started by the portable launcher in 32-bit environments or Force32 is set.
  • Data\TCC_64: contains all 64-bit binaries of TCC/LE and help files. App\TCC_64\tcc.exe is started by the portable launcher in 64-bit environments if 32-bit version is not forced.
  • Optional Data\Plugins_32: contains 32-bit TCC/LE plugins. It is not inherent part of TCC/LE - plugins are loaded by custom 4START script.
  • Optional Data\Plugins_64: contains 64-bit TCC/LE plugins. It is not inherent part of TCC/LE - plugins are loaded by custom 4START script.
